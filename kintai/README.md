# 現場勤怠・人件費管理アプリ セットアップ手順

職人さんが **出先から「現場・氏名・作業内容」を入力** → データが自動集約され、
管理者は **現場ごとの出勤日数を一覧** ＋ **給与情報から人件費を自動計算した Excel** を出力できます。

**LINE公式アカウントやBotは不要です。** 普通のLINEグループに「入力フォームのURL」を貼るだけで運用できます。

```
[管理者] 入力フォームURLを 普通のLINEグループ に貼る（ノートに固定）
                 │
                 ▼  各自タップして開く
[職人A] 入力フォーム ─┐
[職人B] 入力フォーム ─┼─► Firebase(無料DB) ─► [管理者] 集計・Excel出力(admin.html)
[職人C] 入力フォーム ─┘
```

- **入力フォーム**: `index.html`（職人がスマホで使う。氏名は端末に記憶され次回から自動選択）
- **管理画面**: `admin.html`（集計表示・従業員/現場の登録・給与設定・人件費集計・Excel出力。給与は管理者端末にのみ保存）
- **設定ファイル**: `config.js`（Firebaseキー・管理パスコード）

---

## ✅ まず動かしてみる（設定不要・1分）

`index.html` をダブルクリックしてブラウザで開くと、画面と入力操作を確認できます。
（保存先 Firebase が未設定だと「報告」は保存されません。下記Aを設定すると保存できます。）

---

## A. Firebase（データ保存先）の設定 — 約4分

> オセロで使っている Firebase プロジェクトを**そのまま流用してOK**です（データは別パス `kintai/` に
> 保存されるので混ざりません）。`config.js` には既にオセロの値が入っているので、**そのままでも動きます**。
> ただし給与情報を扱うため、本番運用は「専用プロジェクト＋下記ルール」を推奨します（最後の「セキュリティ」参照）。

1. https://console.firebase.google.com/ を開く（流用するなら既存プロジェクトを選択）。
2. 左メニュー **構築 → Realtime Database**（未作成なら「データベースを作成」、ロケーションは任意）。
3. **ルール**タブを開き、`kintai` を読み書き可能にします。既存の `rooms`（オセロ）を残したまま、次のように設定して「公開」：

   ```json
   {
     "rules": {
       "rooms":  { "$room": { ".read": true, ".write": true } },
       "kintai": {
         ".read": true,
         ".write": true,
         "records": { ".indexOn": ["workerName", "date"] },
         "workers": { ".indexOn": ["name"] },
         "sites":   { ".indexOn": ["name"] }
       }
     }
   }
   ```

4. 新しい専用プロジェクトを作った場合のみ、**プロジェクト設定 → マイアプリ（</>）** に表示される
   `firebaseConfig` の値を、`config.js` の `firebase: { ... }` に貼り替えます。
   （オセロを流用するなら変更不要）

これで保存できるようになります。`index.html` で報告 → `admin.html` の「記録一覧」に出れば成功です。

## B. インターネットに公開する（スマホから使うために必須）— 約5分

LINEグループから開けるようにするには、ファイルをネット上に公開してURLを発行します。
オセロと同じGitHubリポジトリに `kintai` フォルダごと上げればOKです。

```powershell
cd "C:\Users\z2000\OneDrive\Desktop\オセロ"
git add kintai
git commit -m "現場勤怠・人件費管理アプリを追加"
git push
```

GitHub の **Settings → Pages** で `main / (root)` が有効なら、数十秒後に次のURLで公開されます：

- 入力フォーム： `https://<あなたのID>.github.io/<リポジトリ名>/kintai/`
- 管理画面　　： `https://<あなたのID>.github.io/<リポジトリ名>/kintai/admin.html`

## C. LINEグループで運用する — 約1分（公式アカウント不要）

1. 職人さんが入っている **普通のLINEグループ** を開く。
2. 上の **入力フォームのURL** をグループに送信。
3. そのURLを長押し →「ノート」に保存、または「アナウンス」に設定して、**上部に固定**しておく。
   - こうすると、メッセージが流れても職人さんがいつでもワンタップで開けます。
4. 職人さんは固定したURLをタップ → フォームが開く → 氏名を選んで報告。
   - **氏名は各自の端末に記憶**されるので、2回目以降は自動で選ばれます。
   - 報告後に表示される **「📲 この報告をLINEグループに貼る」** を押すと、報告内容をそのまま
     グループにも投稿できます（任意。共有先でこのLINEグループを選びます）。

> 管理画面URL（admin.html）は**グループには貼らず**、管理者だけが持っておきます。

---

## D. 使い方

### 職人さん（入力フォーム）
1. グループに固定されたURLをタップ。
2. **氏名 → 作業日 → 現場 → 勤務区分（終日/半日）→（任意で作業時間）→ 作業内容** を入力し「報告する」。
3. 必要なら「LINEグループに貼る」で報告をグループにも共有。
4. 「最近の報告（自分）」で自分の入力履歴を確認できます。

### 管理者（admin.html）
1. パスコード（初期値 `1234`、`config.js` で変更可）を入力。
2. **従業員（給与）タブ**で氏名・**給与方式（日給/時給）・単価**を登録。
   - 入力した給与は**この管理者端末のブラウザに記憶**され、以降の**Excel出力に毎回自動反映**されます（一度入れれば再入力不要）。
   - 給与はクラウド（Firebase）・入力フォームには送られず、**従業員からは見えません**。
   - 別PCへ移すときは同タブの **「⬇ 給与設定をバックアップ(JSON)」/「⬆ 復元」** を使用。
3. **現場タブ**で現場名を登録（フォームのプルダウンに反映）。
4. **集計タブ**で期間を指定すると、**従業員別人件費・現場別人件費**・現場×氏名クロス集計・人件費合計などが自動表示。
5. **「Excelファイルをダウンロード」** で、以下5シートのExcelを出力：
   | シート | 内容 |
   |---|---|
   | 従業員マスタ | 氏名・給与方式・**単価（アプリの設定が自動で入る。Excel上で直すと全シート再計算）** |
   | 勤怠記録 | 全報告の明細＋**1行ごとの人件費（数式）** |
   | 人件費サマリ | 氏名別の出勤日数・実働時間・**人件費（数式）** |
   | 現場別人件費 | **現場ごとの延べ日数・延べ時間・人件費（数式）** |
   | 現場別集計 | 現場×氏名の出勤日数クロス表 |

> **給与・人件費の流れ**：従業員タブで単価を入れておけば、Excelは出力時点で人件費まで計算済みです。
> Excelの「従業員マスタ」シートで単価を直すと、`VLOOKUP`＋`SUMIF` 数式により各シートが自動で再計算されます。

## 🔐 セキュリティ（給与情報の取り扱い）

**給与情報（給与方式・単価）はクラウド（Firebase）や入力フォームには一切送られません。** 給与は管理画面で
入力した**管理者端末のブラウザ（localStorage）にのみ保存**され、集計表示とExcel出力に使われます。従業員の
入力フォームやデータベースには給与が存在しないため、従業員からは見られません。

> 給与は端末ローカル保存のため、ブラウザのデータを消去すると消えます。**「給与設定をバックアップ(JSON)」で
> 控えを保存**しておくと安心です（出力したExcelにも単価は含まれます）。別PCでは「復元」で読み込みます。
> 給与を入れたExcelファイルも、管理者が管理する安全な場所に保管してください。

一方、**勤怠記録・氏名・現場名は共有のため誰でも読み書き可能**な簡易ルールです（給与は含まれません）。
記録自体の保護も強めたい場合は、いずれかを検討してください：

- **専用Firebaseプロジェクトを別に作る**（オセロのゲームデータと分離）。
- **Firebase Authentication** を有効化し、ルールを `".read": "auth != null"` 等に変更。
- `config.js` の `adminPasscode` を初期値 `1234` から変更し、管理画面URLは関係者のみに共有。

困ったら、ブラウザのコンソール（F12）にエラーが出ていないか、`config.js` の `databaseURL` が正しいかを確認してください。
